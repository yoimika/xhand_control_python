项目目录，前面有🐶的是比较关键的文件
	├── __pycache__
	├── camera.py
	🐶├── control_two_hand.py 实时控制两只手，并记录手部数据
	├── data 这是记录了几帧从unity接收到的数据，可以忽略
	🐶├── data_split.py 对接收到的每一帧手部数据进行解析，对于左右手有不同选择方式，可以进行微调
	├── output 采集数据的输出文件夹，可在write.py中修改位置
	├── readme.md
	🐶├── repeat.py 选择手部动作进行重复，可以选择重复执行的指令或实际位置
	├── server.py 服务端代码测试，可忽略
	├── trans.py 四元数计算欧拉角的函数
	├── write.py 记录右手执行动作的每一帧状态，通过xhand失力手动控制
	├── xhand_control_example.py 使用键盘控制xhand每个关节的位置，只能选一只手
	🐶├── xhand_control_left.py 增加了同步关节函数，记录手状态函数
	└── xhand_control_sdk_py 该目录是源码编辑文件夹，大多为C++

采数据过程：
	运行 sudo python control_two_hand.py，左手网线、右手usb
	输入该次数据名称
	即开始采集，包括每个关节位置、温度，和每个指头的触觉信息
	输出在output中对应的文件夹下，包括json文件和两个相机对应的图片

重复某一数据过程
	运行 sudo python repeat.py
	可以通过json中不同的关键字选择重复接收指令还是实际位置
	因此在此可以做到两只手一起重复，只是需要选择重复接收的指令

注意：
1、使用网线进行控制时需要加上sudo，usb控制则不需要
	如sudo python xxx.py
2、出现xxx不存在但又安装过的包，可能是没在root下安装
	sudo pip install 即可
3、使用usb传输手状态信息有点问题，只能受到第一帧的状态，未解决，网线传输无这种情况
4、由于控制信号较长，客户端会发送几个包，服务器端需要等待包接收完全
5、data_split.py处理信号的函数需要和unity发过来的数据形式相对应
6、若服务端没有开启，unity会一卡一卡，因为被tcp协议阻塞

问题：
由于cpu处理速度不够，当记录两个手数据时会导致不能满足最低控制频率，所以会卡顿，只解析一只手的时候不会卡顿，需要解决解析数据比较慢的问题
